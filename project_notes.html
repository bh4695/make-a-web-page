Important Concepts

Lesson 1: The Basics of the Web and HTML

The World Wide Web

The web is a bunch of computers that communicate with each other. HTML documents make up the visual part of the world wide web.

A person receives these HTML documents from the web via HTTP request sent to and from a web browser. Here is a link to a video for further explination.

HTML

HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web.

HTML documents contain text content which describes "what you see" and markup which describes "how it looks". This video gives a good overview.

Tags and Elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags.


Tags: "b" tag (bold) or "em" tag ( italics or emphais). Tags are made up of an open tag and a closed tag with some sort of contents between them. The contents that falls between the open and close tags is what the tags are being applied to.


Attributes: "a" (anchor) attribute is for creating links. This attribute is made up of the attribute name (href), value (web address "www.google.com"), and discripter word that will be highlighted indicating a link is present.


Image: "img" (image) tags are VOID tags, which means they don't have content. They are made up of src (source or location of saved image) and alt (alternate text to be displayed when fails to load.

Communicating with Computers

Communication with computers is very difficult due to the fact that they interpret instructions literally. So the process of communication is very unforgiving due to a small mistake by a programmer can cause huge problems in a program as well as making the debugging process difficult.

Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Container Elements: are elements that contain nested text or content of other elements. Examples of container elements are span which is an inline element and dib which is a block element.

Lesson 2: Creating a Structured Document with HTML

Developer Tools (in the Browser)

Developer Tools allows you inspect or see the HTML elements in a "tree like" structure. This tool also helps the programmer visualize the content as either block or inline.

The "tree-like structure" of HTML

The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. This is also refered to as Nested elements.

In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on.

The relationship between indented HTML and boxes

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested its corresponding "box" is.

Web Page Lanuages

HTML is the structure (framework) of a webpage.
CSS is the style of a webpage, how it looks.
Java Script is the interactive componets of a website.

DOM: Document object Model is the tree like structure of an HTML document.

All elements of an HTML document are rectanglar in shape. So in other words an HTML document is made up of a seris of smaller boxes on top of larger or a large box.

Lesson 3: Adding CSS Style to HTML Structure

Avoiding Repetition

Avoiding repetition is important for a variety of reasons.

Errors: repetition in programming is the act of writing the same code over and over or copy and pasting certain style attributes into many HTML elements many times. This can and in most cases does lead to errors being introduced into your code when a change is made. If the change is is not made everywhere the repeated code appears, problems will arise.

Consistency: by using CSS or STYLE tags it introduces consistency and avoid repetition of code. This is accomplished by assigning div the same class and specifying the style for each class in one place. This allows the programmer to make changes to the style very eaisly and quickly.

CSS

CSS stands for Cascading Style Sheets and they give programmers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class.

Inheritance: (ancestor-decendant relationship) is the mechanism by which properties are applied not only to a specified element, but also its decendents. This means that a nested or child element inherits any properties from the main or parent element unless that property or rule is overridden by property or rule in that specific element.

Code, Test, Refine

Code, Test, Refine refers to the process or steps that programmers take when developing a web page. The 5 main steps are as follows:

1. Look for natural boxes
2. Look for repeated styles and semantiz elements
3. Write your HTML
4. Apply styles (move BIGGEST to smallest)
5. Test and Fix
